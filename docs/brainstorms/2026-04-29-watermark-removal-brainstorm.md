# 去水印 Web 工具设计文档 (Watermark Removal Tool)

## 1. 我们要构建什么 (What We're Building)
一个追求极致性价比、主要用于个人日常处理图片（兼顾免费公开分享）的去水印 Web 工具。
核心交互为：用户上传图片 -> 系统尝试自动识别或用户手动涂抹水印位置 -> 调用第三方图像修复 API 去除水印 -> 用户下载结果。

## 2. 为什么选择这个方案 (Why This Approach)
基于您提到的“不太懂、目前自用也可以、追求性价比”的诉求，我们选择了 **Next.js + Vercel (纯 Serverless)** 方案。
- **零服务器租金**：不需要按月购买昂贵的云服务器，利用 Vercel 平台的免费额度即可将网站免费部署上线。
- **免除复杂运维**：不需要学习复杂的服务器配置，代码提交即自动发布，对新手极其友好。
- **按量计费/免费白嫖**：核心去水印能力调用第三方 API（如阿里云视觉智能、Clipdrop 等），通常有免费调用额度，超出后按次计费，避免了自己部署 AI 模型的高昂闲置成本。

## 3. 关键决策 (Key Decisions)

- **技术栈**：前端与轻量级后端统一使用 Next.js (React) + Tailwind CSS。
  - *验证方式*：后续在 Vercel 部署测试，确保第三方去水印 API 的响应时间在免费版超时限制（10s-15s）内。
- **前端交互与遮罩生成**：使用 HTML5 Canvas（如 `fabric.js` 或 `react-konva`）实现前端画笔涂抹和自动选区功能。
  - *验证方式*：开发一个简单的 Canvas 组件，确认能够导出与原图尺寸一致的黑白遮罩图（Mask）。
- **AI 去水印引擎**：接入成熟的第三方图像处理 API（需支持 Inpainting 图像修补技术 / 允许传入 Mask 遮罩图）。
  - *验证方式*：在正式开发前，调研 1-2 家提供免费额度或计费便宜的 API 服务商并测试效果。

## 4. 不做的范围 (Not Doing)
为了保持项目极简、降低学习门槛和服务器成本，本项目**明确不包含**以下功能：
- **不做用户注册与登录系统**：纯工具形态，打开网页即用。
- **不做云端持久化存储**：用户处理完图片后直接下载到本地（阅后即焚），系统不保存历史记录，节省对象存储空间成本。
- **不做复杂的付费订阅**：目前以自用为主，不涉及微信/支付宝支付接入。

## 5. 遗留问题 (Open Questions)
- **具体 API 选型**：目前市场上提供去水印/修补 API 的厂商较多（如阿里、腾讯、Clipdrop、Photoroom 等），开发前需要最终敲定一家免费额度多或调用单价最低的厂商注册账号并获取 API Key。
